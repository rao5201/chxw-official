   CREATE INDEX IF NOT EXISTS idx_news_published ON news(is_published, date DESC);

### 启用 RLS 行级安全（推荐）

在 SQL Editor 中执行：
```sql
-- 启用新闻表 RLS
ALTER TABLE news ENABLE ROW LEVEL SECURITY;
CREATE POLICY "公开读取新闻" ON news FOR SELECT
  USING (is_published = true);

-- 启用产品表 RLS
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
CREATE POLICY "公开读取产品" ON products FOR SELECT
  USING (is_published = true);

-- 启用工具表 RLS
ALTER TABLE tools ENABLE ROW LEVEL SECURITY;
CREATE POLICY "公开读取工具" ON tools FOR SELECT
  USING (is_published = true);
```

## 3️⃣ 启用分支保护（3 分钟）

### 访问分支设置

1. **进入设置页面**
   ```
   https://github.com/rao5201/chxw-official/settings/branches
   ```

2. **添加保护规则**
   - 点击 **Add branch protection rule**

3. **配置规则**
   ```
   Branch name pattern: main
   ```

4. **勾选保护选项**
   - ✅ **Require a pull request before merging**
     - 禁止直接 push 到 main
   - ✅ **Require approvals**
     - Require 1 approval（至少 1 人审查）
   - ✅ **Dismiss stale pull request approvals when new commits are pushed**
     - 新提交使审查失效
   - ✅ **Require status checks to pass before merging**
     - CI 检查必须通过
   - ✅ **Require branches to be up to date before merging**
     - 分支必须最新
   - ✅ **Require conversation resolution before merging**
     - 解决所有评论
   - ✅ **Include administrators**（可选）
     - 是否适用于管理员
   - ❌ **Allow force pushes**（禁止）
     - 禁止强制推送
   - ❌ **Allow deletions**（禁止）
     - 禁止删除分支

5. **创建规则**
   - 点击 **Create** 按钮

### 效果
- 🔒 main 分支受到保护
- 🔒 必须通过 PR 合并代码
- 🔒 需要代码审查
- 🔒 禁止强制推送和删除
